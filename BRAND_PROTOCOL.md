# 盐究所 NACL-LAB Brand Protocol

## Brand Hierarchy

- `盐究所 NACL-LAB`: 总品牌 / 大中台 / 电子简历。Logo 使用 NACL。
- `红了没`: 小红书产品线，保留做产品字标。
- `爆了没`: 抖音 / 短视频产品线，保留做产品字标。
- `盐究室` / `盐究院`: 其他项目或子品牌，不作为本站总品牌使用。

## Logo Rule

- 公开站点只使用桌面 NACL 资源：`/Users/nacl/Desktop/NΛCL/svg/nacl-logo-white.svg`。
- 站点内复制件放在 `public/assets/brand/nacl-logo-white.svg`。
- 不新画、不改写、不用临时文字 Logo 替代 NACL 主标。

## Product Demo Rule

- `红了没` 和 `爆了没` 当前接入 Coze dev 预览地址。
- dev 预览地址可能休眠；正式发布前必须替换成稳定部署地址。
- 稳定地址由项目负责人提供后再替换，不在代码里猜测。

## Data Discipline

- 不编造价值数字、效率提升、收入、转化率、用户量或业务结果。
- 没有确凿来源的数字一律删除、占位，或明确标注为待确认。
- 私有系统只展示脱敏截图或可公开的结构事实。

## Case Editing Rule

- 案例数据集中维护在 `src/App.tsx` 的 `projects` 数组。
- 新增或修改案例优先只动 `projects` 数组，不为了单个案例改布局。
